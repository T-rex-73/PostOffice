# นโยบายความเป็นส่วนตัว
## ระบบบริหารจัดการงานไปรษณีย์ (PostOffice Management System)

**เวอร์ชัน:** 2.0  
**ปรับปรุงล่าสุด:** มิถุนายน 2569  
**มีผลบังคับใช้:** มิถุนายน 2569

---

## บทนำ

ระบบบริหารจัดการงานไปรษณีย์ ("ระบบ") เป็นเครื่องมือภายในองค์กรสำหรับการลงทะเบียน พิมพ์ซอง และติดตามหนังสือราชการ เราตระหนักถึงความสำคัญของข้อมูลส่วนบุคคลและมุ่งมั่นปฏิบัติตามพระราชบัญญัติคุ้มครองข้อมูลส่วนบุคคล พ.ศ. 2562 (PDPA) นโยบายฉบับนี้อธิบายวิธีที่ระบบเก็บรวบรวม ใช้ และปกป้องข้อมูลของคุณ

---

## 1. ข้อมูลที่เก็บรวบรวม

### 1.1 ข้อมูลบัญชีผู้ใช้
ระบบเก็บข้อมูลต่อไปนี้เมื่อคุณสมัครสมาชิก:

| ข้อมูล | จำเป็น | วัตถุประสงค์ |
|--------|--------|--------------|
| ชื่อ-นามสกุล | ✓ | แสดงตัวตนในระบบ |
| ชื่อผู้ใช้ (Username) | ✓ | เข้าสู่ระบบ |
| รหัสผ่าน (เข้ารหัสแล้ว) | ✓ | ยืนยันตัวตน |
| อีเมล | — | กู้คืนรหัสผ่าน (ถ้ามี) |
| ชื่อสำนักงาน | ✓ | แสดงในเอกสารที่พิมพ์ |
| ตำแหน่งงาน | — | แสดงในระบบ |
| วันที่สมัคร | ✓ | บันทึกอัตโนมัติ |
| วันที่ยอมรับนโยบาย | ✓ | หลักฐานการยินยอม |

### 1.2 ข้อมูลหนังสือราชการ
ข้อมูลที่บันทึกในระบบในฐานะข้อมูลการทำงาน:
- ชื่อ-นามสกุลผู้รับหนังสือ
- ชื่อหน่วยงานและที่อยู่ผู้รับ
- เลขที่หนังสือ
- จังหวัดปลายทาง
- ประเภทไปรษณีย์ (EMS / ธรรมดา)
- ชื่อเจ้าของเรื่อง
- เลขแท็กไปรษณีย์ (Tracking Number)
- วันที่บันทึก

### 1.3 ข้อมูลการใช้งาน
- วันเวลาเข้าสู่ระบบและออกจากระบบ
- การกระทำสำคัญในระบบ (บันทึก แก้ไข ลบ)

---

## 2. วัตถุประสงค์และฐานทางกฎหมาย

| วัตถุประสงค์ | ฐานทางกฎหมาย |
|-------------|--------------|
| ยืนยันตัวตนและควบคุมสิทธิ์เข้าถึง | การปฏิบัติตามสัญญา |
| ให้บริการลงทะเบียนและพิมพ์เอกสาร | การปฏิบัติตามสัญญา |
| บันทึกประวัติเพื่อตรวจสอบย้อนหลัง | ประโยชน์อันชอบด้วยกฎหมาย |
| ปรับปรุงและพัฒนาระบบ | ประโยชน์อันชอบด้วยกฎหมาย |

---

## 3. การเปิดเผยข้อมูลแก่บุคคลที่สาม

ระบบ**ไม่ขาย ไม่ให้เช่า และไม่เปิดเผย**ข้อมูลส่วนบุคคลแก่บุคคลภายนอก ยกเว้นกรณีต่อไปนี้:

### 3.1 ผู้ให้บริการฐานข้อมูล
**Supabase Inc.** ทำหน้าที่ผู้ประมวลผลข้อมูล (Data Processor) โดยข้อมูลถูกจัดเก็บบนเซิร์ฟเวอร์ที่ผ่านมาตรฐาน SOC 2 Type II และ ISO 27001 Supabase ไม่มีสิทธิ์เข้าถึงหรือใช้ข้อมูลเพื่อวัตถุประสงค์อื่นนอกจากการให้บริการโครงสร้างพื้นฐาน

### 3.2 ผู้ดูแลระบบภายในองค์กร
ผู้ดูแลระบบ (Admin) ของหน่วยงานมีสิทธิ์เข้าถึงข้อมูลผู้ใช้และข้อมูลหนังสือทั้งหมดในระบบตามบทบาทหน้าที่ราชการ

### 3.3 ข้อกำหนดทางกฎหมาย
หากมีคำสั่งศาล หน่วยงานที่มีอำนาจตามกฎหมาย หรือความจำเป็นในการปกป้องสิทธิ์ตามกฎหมาย

---

## 4. ความปลอดภัยของข้อมูล

### มาตรการทางเทคนิค
- **รหัสผ่าน**: เข้ารหัสด้วย bcrypt (cost factor 12) ก่อนจัดเก็บ ไม่มีการเก็บรหัสผ่านในรูปแบบ plaintext
- **การสื่อสาร**: ทุกการสื่อสารใช้ TLS 1.2+ (HTTPS)
- **การยืนยันตัวตน**: ผ่าน Supabase Auth พร้อม session token ที่มีอายุจำกัด
- **สิทธิ์การเข้าถึง**: ระบบ Role-Based Access Control (RBAC) แยกระดับสิทธิ์ Admin / User

### มาตรการทางองค์กร
- เฉพาะผู้ที่ได้รับอนุมัติจาก Admin เท่านั้นที่เข้าถึงระบบได้
- ผู้ดูแลระบบรับผิดชอบการจัดการสิทธิ์ผู้ใช้

## 5. การเก็บรักษาข้อมูล

| ประเภทข้อมูล | ระยะเวลาเก็บ |
|-------------|-------------|
| ข้อมูลบัญชีผู้ใช้ | ตลอดอายุบัญชี หรือจนกว่าจะร้องขอลบ |
| ข้อมูลหนังสือราชการ | ตามระเบียบการเก็บเอกสารราชการของหน่วยงาน |
| Log การเข้าสู่ระบบ | 90 วัน |
| ข้อมูลที่อยู่การจัดส่ง | 90 วัน หลังจากนั้นจะลบมูลออกจากระบบ |
| หลักฐานการยินยอม (consent) | ตลอดอายุบัญชี + 1 ปีหลังปิดบัญชี |

## 6. สิทธิ์ของเจ้าของข้อมูล

ในฐานะเจ้าของข้อมูล คุณมีสิทธิ์ดังต่อไปนี้ภายใต้ PDPA:

| สิทธิ์ | รายละเอียด |
|-------|-----------|
| **สิทธิ์รับทราบ** | รับทราบการเก็บรวบรวมและวัตถุประสงค์ |
| **สิทธิ์เข้าถึง** | ขอดูข้อมูลส่วนตัวที่ระบบเก็บไว้ |
| **สิทธิ์แก้ไข** | ขอแก้ไขข้อมูลที่ไม่ถูกต้อง |
| **สิทธิ์ลบ** | ขอลบบัญชีและข้อมูลส่วนตัว |
| **สิทธิ์คัดค้าน** | คัดค้านการประมวลผลในบางกรณี |
| **สิทธิ์ถ่ายโอน** | ขอรับข้อมูลในรูปแบบที่อ่านได้ด้วยเครื่อง |

**วิธีใช้สิทธิ์**: ติดต่อผู้ดูแลระบบของหน่วยงานโดยตรง ระบบจะดำเนินการให้แล้วเสร็จ**ภายใน 30 วัน**นับจากวันรับคำขอ

---

## 7. คุกกี้และการจัดเก็บข้อมูลในเบราว์เซอร์

ระบบใช้:
- **LocalStorage**: สำหรับจดจำ username (ถ้าเลือกตัวเลือก "จดจำรหัสผ่าน") บนอุปกรณ์ของผู้ใช้
- **SessionStorage**: สำหรับส่งข้อมูลชั่วคราวระหว่างขั้นตอนการสมัครสมาชิก (ถูกลบเมื่อปิด tab)
- **ไม่มี** third-party cookies หรือ tracking pixels ใดๆ

---

## 8. การเปลี่ยนแปลงนโยบาย

หากมีการแก้ไขนโยบายนี้ในสาระสำคัญ:
1. ผู้ใช้จะได้รับแจ้งผ่านระบบก่อนที่การเปลี่ยนแปลงจะมีผล
2. วันที่ "ปรับปรุงล่าสุด" ที่ด้านบนของเอกสารนี้จะถูกอัปเดต
3. การใช้งานระบบต่อเนื่องหลังการแจ้งเตือนถือว่ายอมรับนโยบายฉบับใหม่

---

## 9. ติดต่อเจ้าหน้าที่คุ้มครองข้อมูล

หากมีคำถามหรือข้อกังวลเกี่ยวกับนโยบายนี้หรือการประมวลผลข้อมูลของคุณ กรุณาติดต่อ:

**ผู้ดูแลระบบ (System Administrator)**  
ระบบบริหารจัดการงานไปรษณีย์  
ผ่านแชทของระบบ
